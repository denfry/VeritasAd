# evolve-database workflow

## Discovery

- define schema/data change intent and risk class

## Impact Graph

- map affected services, queries, and reports

## Risk Analysis

- evaluate migration duration, lock risk, data correctness

## Compatibility Verification

- use additive-first migration patterns where possible

## Rollback Path

- ensure down migration or compensating rollback plan

## Docs Updates

- update migration guide, upgrade path, and changelog

## Test Expansion

- add migration and data integrity checks

## Telemetry Validation

- monitor query performance and migration health metrics

## Governance Updates

- capture migration lessons into release/debt systems
