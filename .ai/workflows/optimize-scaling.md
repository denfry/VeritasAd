# optimize-scaling workflow

## Discovery

- capture load profile and bottleneck evidence

## Impact Graph

- map bottlenecks across API, queue, workers, DB, storage

## Risk Analysis

- evaluate scaling change risks to reliability and cost

## Compatibility Verification

- preserve external contracts while optimizing internals

## Rollback Path

- define safe rollback to prior capacity profile

## Docs Updates

- update scaling strategy and runbook notes

## Test Expansion

- add load/performance regression checks where feasible

## Telemetry Validation

- compare before/after p95, error rate, queue depth

## Governance Updates

- update scaling thresholds and operational SLO triggers
