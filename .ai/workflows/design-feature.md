# design-feature workflow

## Discovery

- clarify business goal, user journey, and success metric
- map impacted domains, contracts, and dependencies

## Impact Graph

- enumerate touched modules and downstream consumers

## Risk Analysis

- identify compatibility, migration, and performance risks

## Compatibility Verification

- confirm API/event/report contract stability

## Rollback Path

- define revert strategy before implementation

## Docs Updates

- update architecture/requirements/changelog as needed

## Test Expansion

- define unit/integration/contract tests for new behavior

## Telemetry Validation

- ensure new flow emits measurable signals

## Governance Updates

- add new rule/checklist if recurring risk pattern discovered
