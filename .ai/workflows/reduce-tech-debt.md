# reduce-tech-debt workflow

## Discovery

- identify debt class, source, recurrence, and owner

## Impact Graph

- map affected modules and delivery cost impact

## Risk Analysis

- classify severity and business risk

## Compatibility Verification

- ensure debt reduction does not alter required behavior

## Rollback Path

- keep debt paydown changes incremental and reversible

## Docs Updates

- record rationale and outcomes in debt and decision docs

## Test Expansion

- add regression tests for repaid debt areas

## Telemetry Validation

- track defect recurrence rate post-change

## Governance Updates

- convert recurring debt into permanent guardrails
