# prepare-release workflow

## Discovery

- confirm release scope, dependencies, and criticality

## Impact Graph

- enumerate changed modules, contracts, migrations

## Risk Analysis

- assess blast radius and rollback complexity

## Compatibility Verification

- run release checklist compatibility gates

## Rollback Path

- validate rollback instructions and ownership

## Docs Updates

- finalize changelog, release notes, migration and rollback docs

## Test Expansion

- run full required verification suite

## Telemetry Validation

- ensure release monitors and alerts are active

## Governance Updates

- log release lessons into checklists/rules
