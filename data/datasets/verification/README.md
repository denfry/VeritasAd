# Датасет верификации (`data/datasets/verification/`)

Эта директория хранит **фактические данные** датасета верификации (PhD-трек, VeritasAd 3.0), а не документацию. Здесь лежат сами утверждения, доказательства, разметка и артефакты экспериментов; описание схемы и правил разметки вынесено отдельно (см. раздел ниже).

## Структура

```text
claims/        — извлечённые проверяемые утверждения
evidence/      — собранные доказательства (источники, подтверждения/опровержения)
annotations/   — экспертная разметка
experiments/   — артефакты прогонов и оценки
```

## Что коммитится

В Git попадают только лёгкие, версионируемые артефакты:

- `.gitkeep` — чтобы пустые подпапки сохранялись в репозитории;
- этот `README`;
- мелкие `*.jsonl` / `*.json` (сэмплы, схемы, манифесты).

Сырые и крупные файлы **не коммитятся** — они исключены через [`../.gitignore`](../.gitignore). Не добавляйте их в Git вручную: храните вне репозитория или в выделенном хранилище данных.
