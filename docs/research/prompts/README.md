# Prompts

Раздел воркспейса исследований VeritasAd для версионируемых промптов: системные инструкции, шаблоны извлечения и верификации утверждений. Организован по подпапкам в соответствии с треками проекта.

## Подпапки

- [`shared/`](./shared/) — переиспользуемые системные промпты, общие между треками (VeritasAd 2.0, master's и VeritasAd 3.0, PhD). Сюда попадают промпты, не привязанные к конкретной стадии конвейера: общесистемные инструкции, рабочие протоколы, инфраструктурные промпты.
- [`master/`](./master/) — промпты для извлечения утверждений (claim extraction): zero/few-shot извлечение, нормализация, классификация типов, оценка проверяемости (checkworthiness).
- [`phd/`](./phd/) — промпты для верификации (verification): генерация запросов (query generation), оценка достоверности источников (source credibility), определение отношения свидетельства к утверждению (evidence relation), детекция противоречий (contradiction), формирование вердикта (verdict).

## Соглашение об именовании

Имя файла промпта: `<slug>.md`, версия указывается суффиксом `-vN` (например, `claim-extractor-fewshot-v2.md`). При выпуске новой версии создавайте новый файл с инкрементированным суффиксом, не перезаписывая предыдущий, — это сохраняет историю изменений промпта.

Каждый промпт начинается с YAML-фронтматтера по общему соглашению воркспейса (см. шаблон ниже). Для промптов поле `milestone` опускается.

## Примечание о `GEMINI.md`

Файл [`GEMINI.md`](../../../GEMINI.md) **остаётся в корне репозитория**. Это контекст-файл Gemini CLI: инструмент читает его именно из корня, поэтому перенос файла в раздел `prompts/` сломал бы поведение Gemini CLI. Не перемещайте его сюда — ссылайтесь на него относительным путём из корня.

## Шаблон нового промпта

Для создания нового промпта скопируйте шаблон: [`../_templates/prompt.template.md`](../_templates/prompt.template.md). Скопируйте плейсхолдер-значения фронтматтера и заполните их под конкретный промпт.
