# VeritasAd 2.0 — Roadmap магистратуры

**Проект:** VeritasAd  
**Уровень:** магистерская диссертация  
**Базовая линия:** развитие бакалаврского проекта по нейросетевому анализу артефактов рекламного видео  
**Целевой результат:** исследовательская версия VeritasAd 2.0 для извлечения и классификации проверяемых рекламных утверждений

---

## 1. Рекомендуемая тема магистратуры

### Основная формулировка

> **Методы извлечения проверяемых утверждений из рекламного видеоконтента на основе мультимодального анализа и больших языковых моделей.**

### Более строгая формулировка

> **Методы автоматического извлечения и классификации проверяемых рекламных утверждений в мультимодальном цифровом контенте.**

### Практическая формулировка

> **Разработка интеллектуальной подсистемы извлечения проверяемых утверждений из рекламного видеоконтента на основе OCR, ASR и больших языковых моделей.**

### Рекомендуемый вариант для утверждения

> **Методы автоматического извлечения и классификации проверяемых рекламных утверждений в мультимодальном цифровом контенте.**

Причина выбора: формулировка достаточно научная, не сводится только к разработке приложения и оставляет место для экспериментов, сравнения методов и публикаций.

---

## 2. Стратегическая идея

Магистратура должна стать переходом от бакалаврского уровня «система анализирует рекламные признаки» к исследовательскому уровню «система извлекает проверяемые утверждения, которые затем можно верифицировать».

```text
VeritasAd 1.0 — обнаружение рекламных признаков и артефактов
        ↓
VeritasAd 2.0 — извлечение и классификация проверяемых рекламных утверждений
        ↓
VeritasAd 3.0 — доказательная верификация утверждений через внешние источники
```

Магистерская работа должна закрыть именно уровень **VeritasAd 2.0**. Не нужно пытаться в магистратуре полностью решить задачу фактчекинга и анализа противоречивых источников — это лучше оставить для аспирантуры.

## 3. Научное позиционирование

### 3.1. Область исследования

- искусственный интеллект;
- обработка естественного языка;
- мультимодальный анализ цифрового контента;
- анализ рекламного и медиаконтента;
- большие языковые модели;
- OCR и ASR;
- извлечение проверяемых утверждений;
- интеллектуальные информационные системы.

### 3.2. Объект исследования

> **Процессы автоматизированного анализа рекламного и цифрового медиаконтента.**

### 3.3. Предмет исследования

> **Методы извлечения, нормализации и классификации проверяемых рекламных утверждений на основе мультимодальных признаков, OCR, ASR и больших языковых моделей.**

### 3.4. Цель исследования

> **Разработать и экспериментально оценить метод извлечения проверяемых утверждений из рекламного видеоконтента, объединяющий аудиотранскрипты, OCR-текст, брендовые признаки, рекламные маркеры, ссылки, метаданные и большие языковые модели.**

### 3.5. Гипотеза

> **Учет мультимодальных признаков рекламного контента — аудиотранскрипта, OCR-текста, брендовых сигналов, disclosure-маркеров, ссылок и CTA-фраз — повышает качество извлечения проверяемых утверждений по сравнению с анализом только текстовой расшифровки.**

---

## 4. Ожидаемая научная новизна

1. Формализация понятия **проверяемого рекламного утверждения** в мультимодальном цифровом контенте.
2. Классификация рекламных утверждений по типам: количественные, сравнительные, юридические, финансовые, медицинские, субъективные, непроверяемые.
3. Метод извлечения утверждений из рекламного видео на основе объединения ASR, OCR, метаданных и LLM.
4. Метод нормализации рекламных утверждений: преобразование рекламной фразы в строгую проверяемую форму.
5. Модель оценки **checkworthiness** — степени необходимости внешней проверки утверждения.
6. Экспериментальное сравнение text-only, OCR-only, ASR-only, ASR+OCR и full multimodal подходов.

3. Какие типы рекламных утверждений встречаются чаще всего?
4. Как автоматически извлекать claims из аудио, OCR-текста, описаний, ссылок и метаданных?
5. Насколько мультимодальный подход лучше анализа только аудиотранскрипта?
6. Как LLM помогают в нормализации и классификации claims?
7. Какие признаки лучше всего предсказывают checkworthiness?
8. Как подготовить данные для дальнейшей доказательной верификации в аспирантуре?

## 7. Архитектура VeritasAd 2.0

### 7.1. Общая схема

```text
Video / Post / URL
        ↓
Input Processor
        ↓
ASR transcript + OCR text + metadata + links + brand signals
        ↓
Ad Artifact Analyzer
        ↓
Claim Extractor
        ↓
Claim Normalizer
        ↓
Claim Classifier
        ↓
Checkworthiness Scorer
        ↓
Export / API / UI / Dataset
```

### 7.2. Основной новый модуль

Рекомендуемый файл:

```text
backend/app/services/claim_extractor.py
```

Назначение:

- принимать результаты OCR, ASR и анализа рекламных признаков;
- выделять потенциальные утверждения;
- нормализовать рекламную фразу в проверяемую форму;
- классифицировать тип утверждения;
- оценивать проверяемость и риск;
- возвращать структурированный результат.

### 7.3. Пример результата

```json
{
  "claims": [
    {
      "id": "claim_001",
      "raw_text": "Скидка до 70% только сегодня",
      "normalized_claim": "Скидка на товар достигает 70%",
      "source_modality": "ocr",
      "timestamp_start": 12.4,
      "timestamp_end": 14.1,
      "claim_type": "quantitative",
      "is_checkable": true,
      "checkworthiness_score": 0.91,
      "risk_level": "medium",
      "brand": "BrandName",
      "evidence_needed": true
    }
  ]
}
```

## 8. Таксономия рекламных утверждений

| Класс | Описание | Пример |
|---|---|---|
| quantitative | количественное утверждение | «скидка до 70%» |
| comparative | сравнительное утверждение | «быстрее конкурентов» |
| superlative | превосходная степень | «лучший сервис» |
| temporal | утверждение о времени | «доставка за 24 часа» |
| financial | финансовое обещание | «экономия 40%» |
| health_safety | здоровье и безопасность | «безопасно для детей» |
| legal_certification | юридическое или сертификационное утверждение | «сертифицировано» |
| partnership | партнерство или официальный статус | «официальный партнер» |
| availability | наличие или ограниченность | «только сегодня» |
| subjective | субъективная оценка | «идеальный выбор» |
| non_checkable | непроверяемое утверждение | «тебе понравится» |

## 9. Уровни риска claims

| Уровень | Описание |
|---|---|
| low | субъективное или малозначимое утверждение |
| medium | проверяемое рекламное обещание |
| high | финансовое, медицинское, юридическое или безопасностное утверждение |
| critical | потенциально вредное или регуляторно значимое утверждение |

## 10. Checkworthiness scoring

Предлагаемая шкала:

```text
0.00–0.25 — почти не требует проверки
0.26–0.50 — низкая проверяемость
0.51–0.75 — желательно проверить
0.76–1.00 — требует обязательной проверки
```

Признаки для оценки:

- наличие чисел;
- наличие сравнений;
- наличие обещаний результата;
- наличие финансового, медицинского или юридического контекста;
- упоминание сертификатов, гарантий, официального статуса;
- наличие ограничений по времени;
- потенциальный вред пользователю при недостоверности.

## 11. Датасет магистратуры

### 11.1. Минимальный объем

Для защиты желательно иметь:

- 100–300 рекламных видео или постов;
- 500–1500 текстовых фрагментов;
- 300–800 проверяемых утверждений;
- 2 разметчика хотя бы на 20–30% данных;
- train/validation/test split.

### 11.2. Структура данных

```text
data/
  datasets/
    claims/
      raw/
        videos/
        posts/
        screenshots/
      processed/
        transcripts/
        ocr/
        metadata/
      annotations/
        claims_annotation_v1.jsonl
        claims_annotation_v2.jsonl
      splits/
        train.jsonl
        val.jsonl
        test.jsonl
      README.md
```

### 11.3. Формат JSONL

```json
{
  "content_id": "video_001",
  "source_type": "video",
  "source_url": "https://example.com/video",
  "fragment": "Скидка до 70% только сегодня",
  "modality": "ocr",
  "timestamp_start": 12.4,
  "timestamp_end": 14.1,
  "raw_claim": "Скидка до 70% только сегодня",
  "normalized_claim": "Скидка на товар достигает 70%",
  "is_checkable": true,
  "claim_type": "quantitative",
  "risk_level": "medium",
  "brand": "BrandName",
  "annotator_id": "ann_01"
}
```

### 11.4. Инструкция разметчику

Разметчик должен:

1. выделить фрагмент, содержащий утверждение;
2. определить, является ли утверждение проверяемым;
3. нормализовать утверждение;
4. определить тип claim;
5. указать источник: audio, OCR, metadata, link, description;
6. указать риск;
7. отметить неоднозначные случаи.

### 11.5. Контроль качества разметки

- Cohen’s kappa для двух разметчиков;
- процент спорных случаев;
- распределение классов;
- баланс классов;
- доля проверяемых и непроверяемых утверждений.

## 12. Экспериментальный дизайн

### 12.1. Baseline-модели

| Baseline | Описание |
|---|---|
| Rule-based | ключевые слова и регулярные выражения |
| ASR-only | только расшифровка речи |
| OCR-only | только текст на экране |
| ASR + OCR | объединение аудио и OCR |
| BERT/RuBERT classifier | классификация текстовых фрагментов |
| LLM zero-shot | LLM без примеров |
| LLM few-shot | LLM с несколькими размеченными примерами |
| Full VeritasAd 2.0 | мультимодальный метод с признаками рекламы |

### 12.2. Метрики

Для извлечения claims:

- precision;
- recall;
- F1;
- exact match;
- partial match.

Для классификации:

- accuracy;
- macro-F1;
- weighted-F1;
- confusion matrix.

Для checkworthiness:

- ROC-AUC;
- precision@k;
- recall@k;
- correlation с экспертной оценкой.

### 12.3. Ablation study

Проверить варианты:

```text
ASR only
OCR only
ASR + OCR
ASR + OCR + brand signals
ASR + OCR + brand signals + disclosure markers
ASR + OCR + brand signals + disclosure markers + links
Full VeritasAd 2.0
```

### 12.4. Анализ ошибок

Классы ошибок:

- неверное OCR-распознавание;
- шумный ASR;
- неявное рекламное утверждение;
- субъективная фраза ошибочно признана проверяемой;
- проверяемое утверждение пропущено;
- неверная нормализация;
- неверная классификация типа;
- ошибка на сарказме/иронии;
- ошибка на визуальном контексте без текста.

**Тема:** «Методы извлечения проверяемых утверждений из рекламного видеоконтента»

Содержание:

- постановка задачи;
- таксономия claims;
- пилотный датасет;
- baseline-сравнение;
- первые результаты claim extraction.

### Публикация 2

**Тема:** «Мультимодальный подход к выявлению проверяемых рекламных утверждений на основе OCR, ASR и больших языковых моделей»

Содержание:

- сравнение ASR-only, OCR-only и multimodal;
- ablation study;
- checkworthiness scoring;
- анализ ошибок.

## 17. Риски и способы их снизить

| Риск | Что может пойти не так | Как снизить |
|---|---|---|
| Слишком широкая тема | работа превратится в фактчекинг всего интернета | ограничить магистратуру извлечением claims |
| Недостаток данных | нечем доказать качество метода | заранее собрать и разметить корпус |
| Зависимость от GPT API | проблемы воспроизводимости | добавить open-source baseline |
| Слабая научность | получится просто приложение | делать метод, метрики, baseline, ablation |
| OCR/ASR ошибки | claims теряются из-за плохого распознавания | отдельно учитывать качество OCR/ASR |
| Нечеткая разметка | эксперты спорят, что считать claim | подготовить annotation guidelines |
| Большой кодовый объем | много инженерии, мало экспериментов | фиксировать research scope |
| Нет публикаций | слабая траектория к аспирантуре | готовить статью уже со 2 семестра |

## 18. Итоговая цель магистратуры

К окончанию магистратуры VeritasAd должен стать исследовательской платформой:

```text
VeritasAd 2.0 =
Ad artifact detection
+ OCR
+ ASR
+ brand/link/disclosure analysis
+ claim extraction
+ claim normalization
+ claim classification
+ checkworthiness scoring
+ dataset export
+ reproducible experiments
```

Главный научный результат:

> **Разработан и экспериментально подтвержден метод извлечения и классификации проверяемых рекламных утверждений в мультимодальном цифровом контенте.**

Главный задел на аспирантуру:

> **Выделенные и классифицированные claims становятся входом для доказательной верификации через внешние источники, RAG, оценку достоверности источников и анализ противоречий.**
