> **Статус: ЧЕРНОВИК.** Схема JSONL-датасета размеченных проверяемых рекламных утверждений для трека VeritasAd 2.0 (магистратура), веха M3.

## Назначение

Документ описывает схему JSONL для размеченных проверяемых рекламных утверждений. Датасет служит основой для обучения и оценки моделей извлечения и классификации утверждений (claim extraction & checkworthiness): каждая строка файла — отдельное утверждение, привязанное к исходному рекламному контенту (видео, пост, скриншот), с разметкой модальности, типа, уровня риска и проверяемости.

## Структура каталога данных

```text
data/datasets/claims/
  raw/{videos,posts,screenshots}/
  processed/{transcripts,ocr,metadata}/
  annotations/   (claims_annotation_v1.jsonl, claims_annotation_v2.jsonl)
  splits/        (train.jsonl, val.jsonl, test.jsonl)
  README.md
```

Сами данные — в [../../../../data/datasets/claims/README.md](../../../../data/datasets/claims/README.md).

## Формат записи (JSONL, один объект на строку)

```json
{
  "content_id": "video_001",
  "source_type": "video",
  "source_url": "https://example.com/video",
  "fragment": "Скидка до 70% только сегодня",
  "modality": "ocr",
  "timestamp_start": 12.4,
  "timestamp_end": 14.1,
  "raw_claim": "Скидка до 70% только сегодня",
  "normalized_claim": "Скидка на товар достигает 70%",
  "is_checkable": true,
  "claim_type": "quantitative",
  "risk_level": "medium",
  "brand": "BrandName",
  "annotator_id": "ann_01"
}
```

## Описание полей

| Поле | Тип | Описание | Допустимые значения |
| --- | --- | --- | --- |
| `content_id` | string | Идентификатор исходной единицы контента, к которой относится утверждение. | произвольный стабильный id |
| `source_type` | string | Тип источника контента. | `video` \| `post` \| `screenshot` \| `url` |
| `source_url` | string | URL исходного материала. | валидный URL |
| `fragment` | string | Исходный текстовый фрагмент, из которого извлечено утверждение. | произвольный текст |
| `modality` | string | Модальность, из которой получен фрагмент. | `ocr` \| `asr` \| `metadata` \| `link` \| `description` |
| `timestamp_start` | number | Время начала фрагмента в исходном медиа, сек. | ≥ 0 |
| `timestamp_end` | number | Время конца фрагмента в исходном медиа, сек. | ≥ `timestamp_start` |
| `raw_claim` | string | Исходная формулировка утверждения как в контенте. | произвольный текст |
| `normalized_claim` | string | Нормализованная (приведённая к канонической форме) формулировка утверждения. | произвольный текст |
| `is_checkable` | bool | Является ли утверждение принципиально проверяемым. | `true` \| `false` |
| `claim_type` | string | Класс утверждения, см. таксономию. | см. раздел «Таксономия claim_type» |
| `risk_level` | string | Уровень риска утверждения. | `low` \| `medium` \| `high` \| `critical` |
| `brand` | string | Бренд/рекламодатель, к которому относится утверждение. | произвольное название |
| `annotator_id` | string | Идентификатор разметчика, создавшего аннотацию. | произвольный стабильный id |

## Таксономия claim_type

| Класс | Описание | Пример |
| --- | --- | --- |
| `quantitative` | Количественное утверждение. | «скидка до 70%» |
| `comparative` | Сравнительное утверждение. | «быстрее конкурентов» |
| `superlative` | Превосходная степень. | «лучший сервис» |
| `temporal` | Утверждение о времени. | «доставка за 24 часа» |
| `financial` | Финансовое обещание. | «экономия 40%» |
| `health_safety` | Здоровье и безопасность. | «безопасно для детей» |
| `legal_certification` | Юридическое/сертификационное. | «сертифицировано» |
| `partnership` | Партнёрство/официальный статус. | «официальный партнёр» |
| `availability` | Наличие/ограниченность. | «только сегодня» |
| `subjective` | Субъективная оценка. | «идеальный выбор» |
| `non_checkable` | Непроверяемое. | «тебе понравится» |

## Уровни риска

| Уровень | Описание |
| --- | --- |
| `low` | Субъективное/малозначимое. |
| `medium` | Проверяемое рекламное обещание. |
| `high` | Финансовое/медицинское/юридическое/безопасностное. |
| `critical` | Потенциально вредное или регуляторно значимое. |

## Checkworthiness scoring

Каждому утверждению сопоставляется непрерывная оценка приоритета проверки (checkworthiness) в диапазоне от 0.00 до 1.00. Шкала разбита на четыре интервала:

- **0.00–0.25** — почти не требует проверки;
- **0.26–0.50** — низкая необходимость проверки;
- **0.51–0.75** — желательно проверить;
- **0.76–1.00** — требует обязательной проверки.

Оценка опирается на следующие признаки:

- наличие чисел;
- наличие сравнений;
- наличие обещаний результата;
- финансовый/медицинский/юридический контекст;
- упоминание сертификатов/гарантий/официального статуса;
- ограничения по времени;
- потенциальный вред при недостоверности.

## Splits

Датасет делится на три непересекающихся подвыборки в каталоге `splits/`:

- `train.jsonl` — обучающая выборка;
- `val.jsonl` — валидационная выборка;
- `test.jsonl` — тестовая выборка.
