# Планы реализации

Этот раздел содержит **планы реализации milestone'ов** — пошаговые документы, которые переводят дизайн и спецификации в конкретную последовательность действий по коду.

Каждый план описывает, *как именно* будет реализован milestone, и строится по единой структуре:

1. **Context** — зачем нужен этот milestone, что ему предшествует и какие решения уже зафиксированы.
2. **Scope** — что входит в объём работ, а что явно остаётся за его пределами.
3. **Шаги** — упорядоченная последовательность действий по реализации.
4. **Затрагиваемые файлы** — какие модули, конфиги и данные создаются или изменяются.
5. **Риски / rollback** — что может пойти не так и как откатить изменения.
6. **Verification** — как убедиться, что milestone выполнен корректно. Этот блок **зеркалит execution-checklists**: пункты проверки должны соответствовать чек-листам исполнения один к одному.

## Структура раздела

Раздел разделён по трекам исследования:

- [`master/`](master/) — планы для трека VeritasAd 2.0 (магистратура), milestone'ы M1..M5.
- [`phd/`](phd/) — планы для трека VeritasAd 3.0 (PhD), milestone'ы M1..M7.

## Именование файлов

Планы являются датируемыми артефактами и именуются по схеме:

```text
YYYY-MM-DD-<milestone>-<slug>.md
```

Например: `2026-09-01-m2-claim-extractor-mvp.md`. Фаза задаётся папкой (`master/` или `phd/`), milestone — префиксом в имени файла.

## Шаблон

Новый план создаётся копированием шаблона: [`../_templates/plan.template.md`](../_templates/plan.template.md). Шаблон уже содержит YAML-фронтматтер с placeholder-значениями и заготовки всех обязательных разделов — заполните их по месту.
