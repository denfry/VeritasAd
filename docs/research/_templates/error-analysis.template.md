> **Статус: ЧЕРНОВИК.** Скопируйте этот шаблон в файл с датируемым именем `YYYY-MM-DD-<milestone>-<slug>.md` и заполните разделы по конкретному эксперименту.

## Контекст

Опишите, какой эксперимент, модель и набор данных анализируются. Укажите:

- ссылку на эксперимент/отчёт, из которого взяты предсказания;
- версию модели и конфигурацию (промпт, чекпойнт, гиперпараметры);
- срез данных, на котором проводился анализ ошибок (например, валидационная или отложенная выборка), и его размер;
- метрику, по которой отбирались ошибочные примеры (FP/FN, низкий F1 по классу и т. п.).

## Классы ошибок

Стартовая таксономия для трека `master` (извлечение проверяемых утверждений). Заполните столбцы **Частота** и **Пример** по результатам разметки. Удаляйте неактуальные классы и добавляйте новые по мере необходимости.

| Класс | Описание | Частота | Пример |
| --- | --- | --- | --- |
| неверное OCR-распознавание | текст с изображения распознан с ошибками, что исказило извлечённое утверждение | <…> | <…> |
| шумный ASR | ошибки распознавания речи (опечатки, пропуски, склейки слов) исказили вход | <…> | <…> |
| неявное рекламное утверждение | рекламное обещание подразумевается контекстом, но не выражено явным текстом | <…> | <…> |
| субъективная фраза ошибочно признана проверяемой | оценочное/вкусовое высказывание помечено как проверяемое утверждение | <…> | <…> |
| проверяемое утверждение пропущено | реальное проверяемое утверждение не извлечено (false negative) | <…> | <…> |
| неверная нормализация | утверждение извлечено, но нормализовано некорректно (единицы, числа, сущности) | <…> | <…> |

> **Примечание.** Для трека `phd` (анализ verification) используются свои классы ошибок: ошибки relation-классификации, ошибки contradiction-анализа, ошибки оценки credibility. Не смешивайте таксономии master- и phd-треков в одном файле — заводите отдельный артефакт анализа ошибок для каждого трека.

## Примеры по классам

Для каждого значимого класса приведите 2–3 репрезентативных примера: вход (фрагмент текста/OCR/ASR), ожидаемый результат (gold), предсказание модели и краткий комментарий, почему это ошибка именно данного класса.

```text
Класс: <название класса>
Вход:        <фрагмент входных данных>
Ожидалось:   <gold-разметка>
Предсказано: <выход модели>
Комментарий: <почему ошибка отнесена к этому классу>
```

## Гипотезы причин

Сформулируйте проверяемые гипотезы о причинах наиболее частых классов ошибок. Различайте источники:

- данные (шум, дисбаланс классов, ошибки разметки, нехватка примеров);
- предобработка (OCR/ASR, нормализация, сегментация);
- модель/промпт (формулировка инструкции, нехватка контекста, длина окна);
- метрика/протокол оценки (несоответствие критериев gold и предсказания).

## Действия и исправления

Перечислите конкретные действия с приоритетом и ожидаемым эффектом: правки промпта, дообучение, расширение данных, изменение предобработки, корректировка протокола оценки. Свяжите каждое действие с классом ошибок, который оно адресует, и зафиксируйте, как будет измеряться улучшение.
