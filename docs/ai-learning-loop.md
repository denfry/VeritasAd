# AI Learning Loop

## Objective

Continuously convert repeated failures and frictions into repository-level prevention systems.

## Inputs

- repeated bugs
- failed releases
- recurring code smells
- performance regressions
- migration incidents
- security findings
- repeated review comments
- test failures
- architecture drift signals

## Mandatory Transformation Rule

Every repeated issue must become:

1. a new repository rule
2. a workflow update
3. a checklist update
4. a risk pattern entry
5. a prevention strategy with owner

## Loop Cadence

- Weekly: collect signals from PRs, incidents, and CI failures.
- Biweekly: propose updates to docs/workflows/checklists.
- Monthly: prune obsolete rules and measure adoption.

## Quality Guardrails

- Prefer specific, testable rules over broad statements.
- Link each new rule to incident/review evidence.
- Avoid rule bloat by removing redundant controls.
